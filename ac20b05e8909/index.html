LeetCode5: 最长回文子串

题目描述

题目链接

给定一个字符串s，找到s中最长的回文子串，s的最大长度为1000。

  • 示例

题解

暴力匹配（Brute Force）

思路：若字符串长度小于2，直接返回原字符串。否则，枚举所有长度大于等于2的子串来判断是否为回文串，并记录子串起始位置以及子串长度。

动态规划

从回文串的定义展开讨论：

  • 如果一个字符串的头尾两个字符都不相等，那么这个字符串一定不是回文串。
  • 如果一个字符串的头尾两个字符相等，继续向里判断：
    • 如果里面的子串是回文串，整个字符串就是回文串；
    • 如果里面的子串不是回文串，整个字符串就不是回文串。

因此，在头尾字符相等的情况下，里面子串的回文性质决定了整个串的回文性质，这就叫状态转移。对于本题，将状态定义为原字符串的一个子串是否为回文子串。

  1. 定义状态

    dp[i][j]表示子串s[i...j]是否为回文子串，i和j分别表示子串的起始位置和结束位置。

  2. 写出状态转移方程

    动态规划就是在填一张二维表格，在本题中i <= j，当i == j时一个字符构成的字符串是回文串。

    注意i+1和j-1的边界情况。边界条件是区间[i+1, j-1]长度小于2，即 j-1-i-1+1 < 2，j - i < 3。

    当子串s[i...j]的长度等于2或者等于3的时候，只需要判断头尾两个字符是否相等。

    因此，当s[i] == s[j]并且j - i < 3都成立时，可以直接得出dp[i][j] = true，否则就通过状态转移方程确定。

  3. 考虑初始化

    将二维表的对角线初始化为true，即dp[i][i] = true。

  4. 考虑输出

    当得出dp[i][j] = true时，记录此时回文子串的起始位置和回文长度。

⚠️ 填表顺序

每当计算新 dp 值的时候，都一定会参考「左下角」的 dp 值，因为 dp[i + 1][j - 1]（i + 1 表示在下边，j - 1 表示在左边）。

中心扩散法

除了枚举字符串的左右边界以外，比较容易想到的是枚举可能出现的回文子串的“中心位置”，从“中心位置”尝试尽可能扩散出去，得到一个回文串。

因此中心扩散法的思路是：遍历每一个索引，以这个索引为中心，利用“回文串”中心对称的特点，往两边扩散，看最多能扩散多远。

在这里要注意一个细节：回文串在长度为奇数和偶数的时候，“回文中心”的形式是不一样的。

  • 奇数回文串的“中心”是一个具体的字符，例如：回文串 “aba” 的中心是字符 “b”；
  • 偶数回文串的“中心”是位于中间的两个字符的“空隙”，例如：回文串串 “abba” 的中心是两个 “b” 中间的那个“空隙”。

1、如果传入重合的索引编码，进行中心扩散，此时得到的回文子串的长度是奇数；

2、如果传入相邻的索引编码，进行中心扩散，此时得到的回文子串的长度是偶数。

比如传入索引[1, 1]就是以a为中心进行扩散；传入索引[0, 1]就是以b和a之间的间隙进行扩散。

长度为 n 的字符串，有n - 1个间隙和n - 2个中间字符。
