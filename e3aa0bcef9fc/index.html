Java工程师能力要求
2020-09-04 10:57 1.5k字 # 面试 # Java 阅读量：
NEXT: mac下激活Sublime Text3

高级 Java 工程师除了要有技术的广度外，还应该有技术的深度。比如，之前你知道 JVM，知道配置一些参数，但是现在，你需要具备在复杂高并发场景下 JVM 调优的能力，而这一项能力就要求你在深入理解 JVM 的工作原理的同时，还能熟练使用各个分析工具。

以下可当做Java进阶学习路线。

基本能力

各大公司对于高级Java工程师、架构师等高级岗位的能力要求基本都不超过以下几点：

  • Java基础（并发编程、JVM、IO）
  • 开发框架（Spring技术体系、ORM技术体系）
  • 中间件技术（RPC、MQ、Redis）
  • 数据库技术（MySQL优化、主从复制、分库分表）
  • 架构能力（微服务、系统重构、高并发经验）

技术栈

JVM进阶 – Java开发者大厂面试必知必会

  • JVM基础知识、Java字节码技术、JVM类加载器、JVM内存模型、JVM启动参数详解；
  • 常见的JVM GC算法（Parrallel GC/CMS GC/G1 GC）基本原理和特点；
  • 新一代GC算法（Java11 ZGC/Java12 Shenandoah）和Oracle Graalvm；
  • GC日志解读与分析、JVM的线程堆栈等数据分析、内存dump和内存分析工具；
  • fastthread相关工具以及面临复杂问题时几个高级工具的使用；
  • JVM问题排查分析的常用手段、性能调优的最佳实践经验等。

NIO技术 – 构建高吞吐服务器的终极武器

  • 理解NIO相关概念和原理，了解核心知识，以及掌握Netty的技术原理；
  • 同步/异步、阻塞/非阻塞，BIO、NIO、AIO，Reactor/Proactor；
  • ByteBuff/Acceptor/Channel/Handler，NioEventLoopGroup/EventLoop，bossGroup/workerGroup；
  • Netty的启动和执行过程、线程模型、事件驱动、服务端和客户端的使用方式；
  • 常见的API Gateway/HTTP Server，SEDA原理，业务API网关的功能和结构；
  • Throughout/TPS/QPS，Latency/P99/P95/P90，APacheBench/Wrk/Jmeter/LoadRunner。

并发编程 – 多核处理器时代高性能的秘诀

  • Java多线程基础：线程、锁、synchronized、volatile/final、sleep/await/notify/fork/join；
  • Java并发包基础：线程池Executor，AQS/CAS，Atomic原子操作，Lock/Read-WriteLock/Condition，Callable/Future；
  • Java并发容器与工具：BlockingQueue/CopyOnWriteList/ConcurrentHashMap，CountDownLatch/CyclicBarrier/Semaphore等；
  • 万金油ThreadLocal，化繁为简Java8 ParallelStream等。

开发框架 – 深入理解Spring等主流框架思想

  • Spring技术体系（Spring Core/Web/MVC/Data/Messaging、Spring Boot等）；
  • ORM技术体系（JPA、Hibernate、MyBatis等）；
  • 掌握整合各种框架的最佳实践，学会在工作中做技术框架选型；
  • 了解主流框架的发展趋势，一线大厂的使用模式和关注点。

系统性能优化 – 学会性能分析与MySQL优化

  • 系统可观测性（日志、调用链跟踪、指标度量），80/20优化原则，CPU、内存、磁盘/网络IO等分析；
  • MySQL的锁、事务、索引、并发级别、死锁、执行计划、慢SQL统计，缓存失效，参数优化；
  • 库表设计优化、引擎选择、表结构优化设计，列类型选择，索引设计，外键等；
  • SQL查询优化、索引选择、连接优化、聚合查询优化、Union优化、子查询优化、条件优化；
  • 场景分析，主键生成与优化，高效分页，快速导入导出数据，解决死锁问题等。

超越分库分表 – 掌握海量业务数据的应对之道

  • MySQL主从复制，Binlog，Row/Statement模式，主从切换，读写分离，数据库扩容；
  • 数据库垂直拆分和水平拆分，分库分表，分布式主键，分表算法，SQL限制，数据迁移，实时同步；
  • Spring动态切换数据库，TDDL/Sharding-JDBC框架、MyCat/Sharding-Proxy中间件；
  • 数据库拆分的最佳实践，分布式事务最佳实践、多租户的最佳实践。

分布式服务 – 复杂业务系统架构演进必由之路

  • 基础知识：RPC，通信与数据协议，ws，hessian，rest，grpc，protocol buffer等；
  • 服务化：服务治理、配置管理、注册发现，服务分组，版本管理、集群管理，负载均衡、限流与降级熔断；
  • 框架：Apache Dubbo的功能与原理分析，Spring Cloud体系，具体的案例实践；
  • 微服务：深入了解微服务架构的特点和最佳实践，从微服务到服务网格、云原生的介绍。

分布式缓存 – 复杂业务系统访问提速第一法宝

  • 缓存的应用场景，从缓存加载策略、失效策略，缓存与数据库同步等；
  • 缓存预热，缓存失效，缓存击穿，缓存雪崩，多级缓存，缓存与Spring+ORM框架集成；
  • 缓存中间件，Redis（几种常用数据结构，分布式锁，Lua支持，集群），Hazelcast（Java数据结构，内存网格，事务支持，集群）；
  • 缓存的应用场景，排行数据展示，分布式ID生成，Session共享，热点账户操作等。

分布式消息 – 复杂业务系统关系解耦不二法门

  • 消息队列的基本知识，Broker与Client，消息模式（点对点，发布订阅），消息协议（STOMP、JMS、AMQP、OpenMessaging等），消息QoS（最多一次，最少一次，有且仅有一次），消息重试，延迟投递，事务性，消息幂等与去重；
  • 消息中间件：ActiveMQ、RabbitMQ、RocketMQ、Kafka等的基本功能与使用，高可用（集群、分区、副本）；
  • 消息的四个主要功能，搭建一个Kafka集群，实现常用的消息发送、消息消费功能；
  • 典型使用场景，使用MQ实现交易订单的处理，动手实现一个简化版的消息队列。

分布式系统架构 – 如何设计高并发高可用的Java系统

  • 业务分析，功能性需求，非功能性需求，高可用，高性能，稳定性，易用性，扩展性，可维护性，安全性；
  • “4+1”Views，TOGAF，架构方案，业务架构，数据架构，设计文档，技术选型，部署文档，运维文档等；
  • 分布式服务化，分布式消息中间件，分布式缓存，分布式文件系统，监控告警系统，权限与认证中心等。
