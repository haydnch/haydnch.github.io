LeetCode4: 寻找两个正序数组的中位数

题目描述

给定两个大小为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出这两个正序数组的中位数，并且要求算法的时间复杂度为 O(log(m + n))。假设 nums1 和 nums2 不会同时为空。

  • 示例1

    nums1 = [1, 3]

    nums2 = [2]

    则中位数是 2.0

  • 示例2

    nums1 = [1, 3 ,5]

    nums2 = [2, 4，5]

    则中位数是 (3 + 4)/2 = 3.5

题解

分析

题目要求复杂度为O(log(m + n))，因此不能用归并的方法。求中位数，其实就是求第 k 小的数的一种特殊情况，而求第k小的数有一种算法。假设我们要找第k小的数，可以每次循环排除掉k/2个数，看以下例子。

假设要找第7小的数。

先比较两个数组的第k/2个数，如果k是奇数就向下取整。如上图所示，比较两个数组中第3个数。如果哪个小，就表明该数组的前k/2个数都不是第k小的数，所以将这些数排除。在本例中，由于3 < 4，所以排除第二个数组的1，2，3。然后将[1, 3, 4, 9]和[4, 5, 6, 7, 8, 9, 10]两个数组作为新的数组进行比较。

橙色的部分代表被排除的数字。由于已经排除了3个数字。所以在新数组中，只需要找第7 - 3 = 4小的数字，即k变为4。

此时比较第k/2个数，即第2个数。由于3 < 5，所以排除第一个数组中的1和3。

因为排除了两个数字，所以k = 4 - 2 = 2，就是找两个数组中第2小的数字。再比较第k/2小的数，即第1个数。可以看到4==4，由于两个数相等，所以无论排除哪个数组中的4都可以。此时排除掉第二个数组的4。

因为排除了4这一个数，所以k最终减为了1，问题变成了找第1小的数。这时只需比较两个数组中第一个数字哪个小就可以了。因此原问题中第7小的数就是4。

还有一种情况，当某一个数组的长度小于k/2的时候：

此时直接将箭头指向最后一个元素进行比较，由于2 < 3，所以直接将上面的数组直接排除，k = 7 - 2 = 5。

由于长度小于k/2的数组已经清空，因此直接返回下面数组中第5小的数字就可以了。

从上面的例子可以看到，随着算法的进行，k 的值最终都会变成1或者另一个数组变空直接返回结果。

采用递归的思路，为了防止数组长度小于k/2，每次需要比较k/2和len(数组)，选取较小的值对两个数组进行比较。然后排除数较小对应的数组的数，将两个新数组进入递归调用，并且k的值要减去排除的数字个数。递归的出口是当k=1或者其中一个数组的长度为0。
